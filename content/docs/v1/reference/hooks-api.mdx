DocXes provides several custom hooks to interact with the framework's state. These are available for building custom components (e.g., in `x-meta.config.tsx` slot overrides).

## `useSidebar`

Access the global sidebar open/close state. This is useful for creating a sidebar toggle button in your custom header.

### Usage

```tsx
"use client";
import { useSidebar } from "@/components/sidebar-context";

export function ToggleButton() {
  const { toggle, open } = useSidebar();
  
  return (
    <button onClick={toggle}>
      {open ? "Close" : "Menu"}
    </button>
  );
}
```

## `useDocSidebar`

Provides the full navigation tree and expansion logic for the current page.

### Usage

```tsx
"use client";
import { useDocSidebar } from "@/hooks/useSidebar";

export function CustomNav() {
  const { items, expandedItems, toggleExpanded } = useDocSidebar("v1", "/docs/v1/intro");
  // ... render your tree
}
```

### Parameters

<MdxTable 
  headers={["Param", "Type", "Description"]}

    ["`version`", "`string`", "The active documentation version (e.g., \"v1\")."],
    ["`currentPath`", "`string`", "The current URL path for highlighting active items."]
  ]}
/>

## `useDocTOC`

Manages the Table of Contents scroll spy behavior.

### Usage

```tsx
"use client";
import { useDocTOC } from "@/hooks/useTOC";

export function TOC({ headings }) {
  const { activeId } = useDocTOC(headings);
  
  return (
    <ul>
      {headings.map(h => (
        <li className={h.id === activeId ? "active" : ""}>
           <a href={`#${h.id}`}>{h.title}</a>
        </li>
      ))}
    </ul>
  );
}
```
