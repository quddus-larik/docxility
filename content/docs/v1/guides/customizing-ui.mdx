# Customizing UI

DocXes offers three levels of UI customization, ranging from simple configuration tweaks to full component replacement.

## Level 1: Marketplace Themes

The easiest way to change the look of your site is to apply a "Formula" from the marketplace.

```tsx title="x-meta.config.tsx"
import { CurvedUI } from "@/marketplace/curved-ui";

export const XMeta = createConfig({
  ...CurvedUI, // Spread the theme first
  siteName: "My Docs", // Then override specific properties
});
```

Themes are just partial configuration objects. You can inspect `marketplace/curved-ui.tsx` to see exactly what it changes (usually `cssVars` and some `styles` overrides).

## Level 2: Style Overrides

If you want to keep the standard components but tweak their CSS, use the `styles` property. This allows you to append Tailwind classes to specific parts of the component.

```tsx
sidebar: {
  styles: {
    // Make the sidebar background blue and add a border
    sidebar: "bg-blue-50 border-r-2 border-blue-200",
    
    // Style the navigation items
    item: "text-blue-900 hover:bg-blue-100",
    
    // Style the active item
    itemActive: "font-bold bg-blue-200",
  }
}
```

## Level 3: Component Slots

For complete control, you can replace a section of the layout with your own React component.

### Example: Custom Sidebar Header

Let's add a company logo and a version picker to the sidebar.

```tsx title="x-meta.config.tsx"
import { VersionSelect } from "@/components/version-select";

export const XMeta = createConfig({
  sidebar: {
    header: ({ version, versions }) => (
      <div className="p-4 border-b bg-muted/30">
        <div className="flex items-center gap-2 mb-3">
          <img src="/logo.svg" className="h-6 w-6" />
          <span className="font-bold">Acme Corp</span>
        </div>
        
        {/* Use the client component for interactivity */}
        <VersionSelect version={version} versions={versions} />
      </div>
    )
  }
});
```

<MdxAlert type="warning" title="Client Components">
  The `x-meta.config.tsx` file runs in a Server Component context. If you need interactivity (like `onClick` handlers, `useState`, or `useEffect`), you **must** create a separate component file with `"use client"` at the top and import it into your config.
</MdxAlert>
