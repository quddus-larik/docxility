# Configuration Guide

The `x-meta.config.tsx` file is where you control the behavior and appearance of your DocXes site.

## The `createConfig` Factory

We use a factory function to ensure type safety and allow for "partial" configurations (like Themes) to be merged easily.

```tsx
import { createConfig } from "@/lib/configuration";

export const XMeta = createConfig({
  // options...
});
```

## Options Reference

### `siteName`
- **Type**: `string`
- **Default**: "DocXes"
- **Description**: The name of your site. Used in the page title (`<title>`) and the default header logo.

### `description`
- **Type**: `string`
- **Default**: "A dynamic documentation generator framework"
- **Description**: Used for SEO metadata description.

### `siteUrl`
- **Type**: `string`
- **Description**: The production URL of your site (e.g., `https://docs.myapp.com`). Critical for generating correct canonical tags and sitemaps.

### `documentsPath`
- **Type**: `string`
- **Default**: `content/docs`
- **Description**: The file system path where your MDX files are stored.

### `searchProvider`
- **Type**: `"local" | "flexsearch"`
- **Default**: `local`
- **Description**: The search engine to use.

### `theme`
- **Type**: `object`
- **Description**: Controls visual styling.
  - `mdx.highlighter`: `"pretty-code"` (default) or `"none"`.
  - `mdx.theme`: The Shiki theme name (e.g., `github-dark`, `dracula`).
  - `mdx.keepBackground`: `boolean`. If `false`, code blocks use the container's background color.
  - `cssVars`: The Headless Tailwind configuration object.

### `versions`
- **Type**: `object`
- **Description**: Versioning behavior.
  - `default`: The version to redirect to when visiting the root `/docs` path (e.g., `"v1"`).

## Slot Overrides

These properties allow you to replace core layout components.

- `header`: Replace the top navigation bar.
- `footer`: Replace the site footer.
- `sidebar.component`: Replace the sidebar.
- `sidebar.header`: Insert content above the sidebar navigation.
- `sidebar.footer`: Insert content below the sidebar navigation.
- `toc.component`: Replace the Table of Contents.
- `pagination.component`: Replace the Next/Prev buttons.
