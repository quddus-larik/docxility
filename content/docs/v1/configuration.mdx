# Configuration

The `x-meta.config.tsx` file is the heart of your DocXes site. It uses a type-safe configuration factory to define metadata, theme behaviors, and component overrides.

## Basic Settings

These settings define your site's identity and global behavior.

| Property | Type | Description |
| :--- | :--- | :--- |
| `siteName` | `string` | The name shown in the header and metadata. |
| `siteUrl` | `string` | The base URL for canonical tags and sitemaps. |
| `searchProvider` | `"local" \| "flexsearch"` | Select the engine for documentation search. |
| `documentsPath` | `string` | Path to your content folder (default: `content/docs`). |

## Theme Configuration

DocXes uses a headless approach to theming. You can configure MDX behavior and global styles.

### MDX Options

```tsx
theme: {
  mdx: {
    highlighter: "pretty-code", // Syntax highlighter
    theme: "github-dark",       // Shiki theme name
    keepBackground: false,      // Use container background for code blocks
  }
}
```

### Headless Tailwind (`cssVars`)

DocXes injects CSS variables into your site dynamically. You can change your primary colors and border radius without editing CSS files.

```tsx
theme: {
  cssVars: {
    root: { "radius": "0.75rem" },
    light: {
      "primary": "oklch(0.5 0.2 250)",
      "background": "oklch(1 0 0)",
    },
    dark: {
      "primary": "oklch(0.9 0.05 250)",
      "background": "oklch(0.1 0 0)",
    }
  }
}
```

## Marketplace Themes

Instead of building from scratch, you can use pre-defined formulas from the `marketplace/` directory.

```tsx
import { ShadcnUI } from "@/marketplace/shadcn-ui";
import { CurvedUI } from "@/marketplace/curved-ui";

// Choose one and spread it:
export const XMeta = createConfig({
  ...CurvedUI, 
  siteName: "Curved Docs",
});
```

## Component Slots

You can inject your own components or logic into specific areas of the layout.

```tsx
sidebar: {
  // Add a logo or custom title to the sidebar
  header: ({ version, versions }) => (
    <div className="p-4 border-b">
      <Logo />
      <VersionSelect version={version} versions={versions} />
    </div>
  ),
  // Customize individual sidebar items
  item: MyCustomSidebarItem,
},
toc: {
  // Add a footer to the Table of Contents
  footer: () => <div className="p-4">Helpful Links</div>
}
```

<MdxAlert title="Interactivity Note" type="warning">
  If you pass a function to a slot (like `sidebar.header`), ensure any interactive elements (like inputs or dropdowns) are wrapped in a **Client Component** (using `"use client"`).
</MdxAlert>
