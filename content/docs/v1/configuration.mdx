# Configuration Guide

The `x-meta.config.tsx` file is the control center of your documentation site. It handles metadata, theming, UI overrides, and feature toggles.

## Basic Metadata

These fields control the SEO and general identity of your site.

```typescript
export const XMeta = {
  siteName: "DocX Framework",
  description: "Production-ready documentation system.",
  siteUrl: "https://your-domain.com",
  documentsPath: "content/docs", // Where your content lives
  // ...
};
```

## Theming & Headless Tailwind

DocX allows you to customize the look and feel **without** writing CSS files. We inject CSS variables at runtime based on your config.

### MDX Theme

Control the syntax highlighting style.

```typescript
theme: {
  mdx: {
    highlighter: "pretty-code",
    theme: "github-dark", // or 'github-light', 'dracula', etc.
    keepBackground: false,
  },
},
```

### CSS Variables (Headless Customization)

You can override Tailwind's base variables directly. This updates colors, border radius, and spacing globally.

```typescript
theme: {
  cssVars: {
    // Applied to :root
    root: {
      "radius": "0.5rem", // Changes rounded-lg, rounded-md, etc.
    },
    // Applied when data-theme="light" or system light preference
    light: {
      "primary": "oklch(0.21 0.006 285.885)", // Deep Blue
      "primary-foreground": "oklch(0.985 0 0)", // White
      "background": "oklch(1 0 0)",
    },
    // Applied when class="dark"
    dark: {
      "primary": "oklch(0.985 0 0)", // White
      "primary-foreground": "oklch(0.21 0.006 285.885)", // Deep Blue
      "background": "oklch(0.141 0.005 285.823)",
    }
  }
}
```

## Interface & Plugins

DocX uses a slot-based architecture. You can swap out core components or just inject content into headers and footers.

### Component Slots

```typescript
interface: {
  components: {
    // Inject Custom Header into Sidebar
    sidebarHeader: ({ version }) => (
      <div className="font-bold p-4">My Docs {version}</div>
    ),
    
    // Inject Custom Footer into TOC
    TOCFooter: () => <a href="#">Edit this page</a>,
    
    // Completely replace the Sidebar component
    sidebar: CustomSidebarComponent, 
  }
}
```

### Style Overrides

You can also pass custom Tailwind classes to core components to tweak their appearance without replacing them.

```typescript
interface: {
  styles: {
    sidebar: {
      sidebar: "bg-zinc-900 border-r border-zinc-800", // Dark sidebar
      itemActive: "text-blue-500 bg-blue-500/10", // Active link style
    },
    TOC: {
      title: "text-blue-500 font-mono", // Custom font for TOC title
    },
    pagination: {
      button: "hover:border-blue-500", // Custom hover effect
    }
  }
}
```
