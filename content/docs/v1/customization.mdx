# Custom UI Components

For maximum flexibility, DocXes allows you to replace entire UI sections with your own React components or arrow functions.

## Using Arrow Functions

You can define custom headers, footers, and item renderers directly in your config using arrow functions. These functions receive default parameters to help you stay in sync with the framework.

### Custom Sidebar Header

```tsx
sidebar: {
  header: ({ version }) => (
    <div className="p-6 border-b">
      <h2 className="text-xl font-bold">My Project</h2>
      <span className="text-xs bg-primary text-white px-2 py-0.5 rounded">
        v{version}
      </span>
    </div>
  ),
}
```

### Custom Page Header

The main site header can also be customized:

```tsx
header: ({ siteName, versions }) => (
  <header className="flex justify-between items-center p-4 bg-white shadow">
    <Logo />
    <nav>
      {versions.map(v => <a key={v} href={`/docs/${v}`}>{v}</a>)}
    </nav>
  </header>
)
```

## Component Slots

DocXes uses a slot-based architecture. You can override the default component for major sections:

- `sidebar.component`
- `toc.component`
- `pagination.component`
- `header`
- `footer`

By providing your own component to these slots, you can completely redefine the user experience while still benefiting from DocXes's data fetching and state management hooks.

### Hooks for Custom Components

If you build your own components, you can use our built-in hooks:

- `useDocSidebar(version, currentPath)`: Get navigation items and mobile state.
- `useDocTOC(headings)`: Get active heading and scroll logic.
- `useContentData(version, slug)`: Get the full page data.
