# Customization

DocXes is designed to be fully extensible. You can go beyond simple configuration and build your own logic using slots, hooks, and custom components.

## Slot-Based Architecture

Every major section of the layout is a "slot." You can override these in `x-meta.config.tsx`.

### Core Layout Slots

| Slot | Description |
| :--- | :--- |
| `header` | The global site header. |
| `footer` | The global site footer. |
| `sidebar.component` | The entire navigation sidebar. |
| `sidebar.header` | Content above the navigation links. |
| `sidebar.footer` | Content below the navigation links. |
| `toc.component` | The "On this page" sidebar. |
| `pagination.component` | The bottom "Next/Prev" buttons. |

### Overriding with Arrow Functions

Simple overrides can be done directly in the config:

```tsx
sidebar: {
  header: ({ version, versions }) => (
    <div className="p-4 bg-muted/50 rounded-lg m-4">
      <h3 className="font-bold">Project {version}</h3>
      <VersionSelect version={version} versions={versions} />
    </div>
  )
}
```

## Built-in Hooks

If you build your own custom components for these slots, you can leverage our internal hooks to access framework state.

### `useDocSidebar`
Returns navigation items, mobile open state, and expansion logic.
```tsx
const { items, open, setOpen, toggleExpanded } = useDocSidebar(version, currentPath);
```

### `useDocTOC`
Returns the active heading based on the current scroll position.
```tsx
const { activeId } = useDocTOC(headings);
```

### `useContentData`
(Server Side) Retrieves the full data object for a specific document, including metadata, content, and breadcrumbs.

### `useSidebar`
(Client Side) Accesses the global sidebar state (useful for creating toggle buttons in your own header).
```tsx
const { toggle, open, setOpen } = useSidebar();
```

## Creating Custom Search Providers

DocXes abstracts search behind the `SearchProviderInterface`. You can implement your own provider (e.g., for Algolia) by creating a class that implements:

```typescript
interface SearchProviderInterface {
  name: string;
  initialize(): Promise<void>;
  search(query: string, options?: SearchOptions): Promise<SearchResult[]>;
}
```

Then, update your config:
```tsx
searchProvider: new MyAlgoliaProvider(),
```
