# Components & MDX

DocX comes with a suite of built-in React components that you can use directly in your `.mdx` files.

## Standard Markdown

All standard Markdown syntax is supported:

- **Bold**: `**text**`
- _Italic_: `*text*`
- [Links](https://example.com)
- Lists (ordered and unordered)
- Tables
- Blockquotes

## Built-in Components

### Alerts

Use `MdxAlert` to call attention to important information.

```jsx
<MdxAlert type="info" title="Note">
  This is a useful piece of information.
</MdxAlert>

<MdxAlert type="warning" title="Caution">
  Be careful when modifying this setting.
</MdxAlert>

<MdxAlert type="error" title="Critical">
  Do not delete the production database.
</MdxAlert>

<MdxAlert type="success" title="Done">
  Setup completed successfully.
</MdxAlert>
```

### Cards

Cards are great for grouping links or concepts.

```jsx
<MdxCard title="Getting Started" href="/docs/v1/getting-started">
  Learn the basics of DocX.
</MdxCard>
```

### Badges

Inline badges for versioning or status.

```jsx
This feature is <MdxBadge>Beta</MdxBadge> and available in <MdxBadge variant="secondary">v2.0</MdxBadge>.
```

### Steps

Use `StepsWithCounter` and `Step` to document tutorials.

```jsx
<StepsWithCounter>
  <Step>
    #### Install Node.js
    Ensure you have Node v18+ installed.
  </Step>
  <Step>
    #### Clone Repo
    `git clone ...`
  </Step>
</StepsWithCounter>
```

### Code Blocks

Code blocks support syntax highlighting and filenames.

```typescript title="example.ts"
function hello(name: string) {
  console.log(`Hello ${name}`);
}
```

## Adding Custom Components

To add your own components:

1. Create the component in `components/mdx/`.
2. Register it in `components/mdx/mdx-components.tsx`.
3. It will be available globally in all MDX files.
