# Components

DocXes provides a set of high-quality components designed for technical documentation. These are available globally in all your `.mdx` files.

## Alerts (`MdxAlert`)

Use alerts to provide contextual feedback or emphasize important points.

<MdxAlert type="info" title="Info">
  Standard information or helpful tips.
</MdxAlert>

<MdxAlert type="warning" title="Warning">
  Actions that require caution.
</MdxAlert>

<MdxAlert type="success" title="Success">
  Confirmation of a successful action.
</MdxAlert>

<MdxAlert type="error" title="Error">
  Critical issues or dangerous operations.
</MdxAlert>

```jsx
<MdxAlert type="info" title="Important">
  This is an alert.
</MdxAlert>
```

## Cards (`MdxCard`)

Cards are ideal for grouping related links or highlighting key sections.

<MdxCard title="Architecture" href="/docs/v1/architecture">
  Deep dive into the framework's core.
</MdxCard>

```jsx
<MdxCard title="Card Title" href="/docs/v1/page">
  Card content goes here.
</MdxCard>
```

## Steps (`StepsWithCounter` / `Step`)

Perfect for tutorials and installation guides.

<StepsWithCounter>
  <Step>
    #### Step One
    Install the package.
  </Step>
  <Step>
    #### Step Two
    Configure the API key.
  </Step>
</StepsWithCounter>

```jsx
<StepsWithCounter>
  <Step>
    #### First Step
    Instructions...
  </Step>
</StepsWithCounter>
```

## Badges (`MdxBadge`)

Small status indicators.

<div className="flex gap-2 mb-4">
  <MdxBadge>Default</MdxBadge>
  <MdxBadge variant="secondary">Secondary</MdxBadge>
  <MdxBadge variant="outline">Outline</MdxBadge>
  <MdxBadge variant="destructive">Destructive</MdxBadge>
</div>

```jsx
<MdxBadge variant="secondary">Beta</MdxBadge>
```

## Code Blocks

Code blocks are automatically enhanced with:
- **Syntax Highlighting** (via Shiki/rehype-pretty-code)
- **Copy to Clipboard** button
- **Line Highlighting**
- **Word Highlighting**

```typescript title="lib/utils.ts" {1-2}
export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```
